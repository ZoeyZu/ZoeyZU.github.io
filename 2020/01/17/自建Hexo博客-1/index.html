静态博客Hexo搭建 —— 本地、服务器两开花

静态博客 —— Hexo
静态博客：是指没有后端数据库的博客，所有文章都是以文件的形式存储，是一种纯前端的博客框架。

本地安装

Step1：安装node.js

不停next即可

  • Hexo是通过Node.js生成的，所以首先需要根据自己的操作系统版本下载并安装Node.js。
  • 根据步骤提示进行操作后，在终端中输入命令：node -v。如果能正确显示Node.js版本，则说明安装成功（本人使用的版本为v10.15.3）。

Step2：安装Git

不停next即可

Step3：安装Hexo

Node.js安装成功后，同时会自动安装一个包管理器，称为npm（Node Package Manager）。

什么是npm？

  • npm是一个包管理器，发布者将自己的代码发布（npm publish）到代码仓库（registry），使用者将自己需要的包写入package.json文件中，到代码仓库中把代码下载（npm install）下来。
  • 安装：npm install -g hexo-cli
  • 查看版本：hexo -v

由于npm代码仓库的服务器在国外，下载速度不尽如人意，可以使用淘宝的npm代码仓库，通过npm安装cnpm：
npm install -g cnpm --registry=https://registry.npm.taobao.org
安装成功后，可以通过以下命令查看cnpm版本：
cnpm -v
通过cnpm来操作下载速度会得到很大提升，但包的版本不一定是最新的。

==完成之后安装hexo==

  • cnpm install -g hexo-cli
  • 验证是否安装成功：hexo -v

本地使用

运行hexo s
打开浏览器，输入localhost:4000

经常用到的几个命令

1. 创建目录

建立一个空的文件夹，在终端打开文件目录：sudo hexo init

Hexo的所有文章都是以文件的形式存储在目录下的，所以首先需要新建一个目录，我的是Hexo，在此目录下操作。

2. 初始化博客

hexo init

3. 新建文章

hexo new "BlogName"（BlogName是你的文章名称）

会在source/_posts目录下生成BlogName.md文件，可以vim或其他Markdown编辑器进行修改。

4. 生成博客静态文件

  • hexo g（g代表generate），生成博客静态文件
  • 每次修改文章后，都需要通过hexo clean清理一下，
  • 然后通过hexo g重新生成，
  • 最后也不要忘了通过hexo s重新启动Hexo。

5. 启动博客

hexo s（s代表server，默认端口号是4000）

6. 更换主题

  • 将需要更换的主题下载并解压缩到theme/ThemeName目录下，ThemeName是主题名字，
  • 也可以直接在Hexo目录下执行git clone https://github.com/theme-next/hexo-theme-next themes/next，
  • 解压完成后，修改_config.yaml文件中的theme属性，默认是landscape，修改为next
  • 每次修改主题后，都需要通过hexo clean清理一下，
  • 然后通过hexo g重新生成，
  • 最后也不要忘了通过hexo s重新启动Hexo。

服务器部署

本地 + github.io 白嫖部署

1. 生成github.io仓库

  • 首先注册登录GitHub
  • 创建新public仓库，仓库名称一定要是：GtHubName.github.io（GitHubName是本人的GitHub昵称，大小写敏感！）

2. 本地安装Hexo的git部署插件

在Hexo的目录下，输入npm install --save hexo-deployer-git，会报一个peerDependencies WARNING，可以忽略。
